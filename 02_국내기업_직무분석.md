# 1. 더존비즈온

---

## ① B2B IT 솔루션 영업

### 주요 업무 및 책임

- ERP(iCUBE, WEHAGO, Amaranth 10), 그룹웨어, 클라우드 서비스의 신규 고객 발굴 및 계약 체결
- 기존 고객사 유지 관리, 추가 모듈 업셀링(Upsell), 갱신 계약(Renewal) 협상
- 담당 지역(서울·경기·인천·부산·대구·광주 등 전국 권역) 기업 대상 방문 영업
- 고객사의 경영·업무 프로세스 분석 → IT 솔루션 도입 필요성 제안
- 영업 파이프라인 관리 (CRM 입력·보고)
- 내부 개발·컨설팅 팀과 협업하여 POC(개념증명) 및 구축 프로젝트 지원

### 요구 역량

#### 기술적 역량

- ERP 기본 개념 이해: 회계·인사·급여·물류 모듈 흐름
- 제품 데모 능력: iCUBE, WEHAGO, Amaranth 10, OmniEsol 시연
- MS Office(Excel 고급) 및 CRM 도구 활용
- 운전면허 2종 보통 이상 (지방 출장 필수)
- 클라우드·SaaS 비즈니스 모델 이해

#### 비기술적 역량

- 적극적 성향, 강한 목표 지향성
- 고객 신뢰 구축 능력 및 장기적 관계 관리
- 설득력 있는 프레젠테이션 및 커뮤니케이션
- 일 단위 실적 압박 환경 내성 (현직자 리뷰: "일 단위 실적 보고, 목표 미달 시 즉각 피드백")
- 내부 개발자·컨설턴트 설득 역량 (내부 영업도 병행)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 월별·분기별 신규 계약 매출 목표 달성률 | 목표 대비 % |
| 신규 고객 확보 건수 | 월 단위 목표치 설정 |
| 갱신율(Renewal Rate) / 업셀링 매출 비중 | % |
| 파이프라인 규모 및 Win Rate | 계약 성사율 25% 이상 목표 |
| POC 이후 정식 계약 전환율 | % |
| 영업 활동량 KPI | 방문 미팅 건수, RFP 제출 건수 |

### 직무별 도전과제

- **일 단위 실적 압박 문화:** 전월 300% 달성 후에도 당월 목표 미달 시 불이익 (잡플래닛·블라인드 현직자 복수 언급)
- **내부 협업 장벽:** 개발자/컨설턴트 미협조 시 영업 자체가 불가능
- **제품 포트폴리오 복잡성:** ERP 솔루션이 기업 규모(소기업→대기업)별로 다원화되어 있어 솔루션 매핑 어려움
- **클라우드 전환 압박:** 기존 온프레미스 고객의 클라우드(WEHAGO) 전환 설득
- **긴 의사결정 주기:** B2B ERP 도입 결정은 평균 60~120일 소요

## ② EMS/AIOps 소프트웨어 개발자

### 주요 업무 및 책임

- WEHAGO / Amaranth 10 / OmniEsol 플랫폼의 백엔드(Java·Spring 기반) 신규 기능 개발 및 유지보수
- ERP 모듈(회계·인사·급여·물류) API 설계 및 구현
- 생성형 AI(GenAI) 기반 기능 개발 (DEWS 플랫폼 내 AI 접목)
- 클라우드 데이터센터(D-클라우드센터) 연동 시스템 개발
- 코드 리뷰, 단위/통합 테스트, CI/CD 파이프라인 관리
- 성능 최적화 및 대용량 데이터 처리 구조 설계
- 빅데이터 기반 ERP 분석 기능 개발 (WEHAGO 빅데이터 모듈)

### 요구 역량

#### 기술적 역량

| 기술/도구 | 수준 |
|---|---|
| Java, Spring MVC/Boot | 필수 |
| MySQL(RDBMS), Linux | 필수 |
| JavaScript, Vue.js, React, HTML5/CSS3 | 프론트 협업 |
| CI/CD 파이프라인, Docker, Git | 필수 |
| ERP 도메인 지식, 전자정부 표준프레임워크 | 우대 |
| Python, GenAI 연동 API | 우대 (OmniEsol 관련) |

#### 비기술적 역량

- 대규모 레거시 코드베이스 분석·개선 의지
- 비즈니스 도메인 이해: 회계·세무·인사 프로세스 학습 능력
- 영업·컨설팅 팀과의 긴밀한 협업 커뮤니케이션
- 프로젝트 일정 준수 책임감

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 스프린트 완료율 | 계획 대비 실제 완료 스토리 포인트 비율 |
| 배포 빈도 및 배포 성공률 | 주기적 릴리즈 기준 |
| 결함 밀도 | 출시 후 버그 발생 건수 / 기능 규모 |
| 변경 실패율 (DORA 지표) | 배포 후 롤백·긴급패치 비율 |
| 코드 리뷰 참여율 및 피드백 반영률 | % |
| 기능 납기 준수율 | 요구사항 확정 → 출시 리드타임 |

### 직무별 도전과제

- **레거시 ERP 코드베이스:** 20년 이상 축적된 레거시 코드 유지보수와 신기술 도입 병행의 기술 부채
- **ERP 도메인 학습 곡선:** 회계·세무·인사 등 복잡한 비즈니스 로직 이해 필요
- **클라우드 전환 가속:** 온프레미스 → SaaS 아키텍처 전환 작업 동시 진행
- **AI 통합 요구 증가:** OmniEsol의 GenAI 기능 개발 요구가 빠르게 증가
- **멀티 플랫폼 지원:** WEHAGO, Amaranth 10, iCUBE 등 복수 플랫폼 병렬 개발

## ③ IT 인프라 운영 엔지니어

### 주요 업무 및 책임

- D-클라우드센터(자체 IDC) 서버·네트워크 365×24 모니터링 및 관제
- 고객사·그룹사 서버 시스템 인프라 구축 및 운영 (Linux, Windows)
- 장애 감지·분석·처리·복구 (트러블슈팅)
- 가상화 솔루션 운영: VMware, Hyper-V 기반 VM 관리
- 그룹웨어(WEHAGO 등) 시스템 접속 장애 및 오류 대응
- 백업 시스템, 백신, 부가서비스 운영 지원
- 인프라 성능 점검 및 인증심사 지원 (보안 인증, 클라우드 인증 등)
- 서버 하드웨어 교체 및 OS/앱 업데이트 관리

### 요구 역량

#### 기술적 역량

| 기술/도구 | 수준 |
|---|---|
| Linux/Windows 서버 운영 | 필수 |
| 네트워크 기초 (TCP/IP, VLAN, 방화벽) | 필수 |
| VMware, Hyper-V (가상화) | 필수 |
| 서버 하드웨어: RAID 구성, 파티셔닝 | 필수 |
| Zabbix, Nagios 계열 모니터링 도구 | 우대 |
| 정보처리기사·산업기사 자격증 | 우대 |
| Bash, Python (자동화 스크립팅) | 우대 |

#### 비기술적 역량

- 높은 책임감: 사소한 실수도 대규모 서비스 장애로 이어질 수 있는 환경
- 야간·주말 교대 근무 수용 (365×24 관제 체계)
- 능동적 사고: 수동 대응이 아닌 선제적 이슈 발견 및 개선안 제시
- 근무지 인근 거주 우대 (춘천 IDC 센터 기준)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 목표 |
|---|---|
| 시스템 가용성 (Uptime) | SLA 기준 99.9% 이상 유지율 |
| 장애 평균 복구 시간 (MTTR) | 최소화 목표 |
| 장애 평균 발생 간격 (MTBF) | 최대화 목표 |
| 고객 지원 요청 처리율 및 처리 속도 | 티켓 해결률 |
| 예방점검 완료율 | 정기 점검 계획 대비 실행률 |
| 보안 취약점 패치 적기 적용률 | % |

### 직무별 도전과제

- **야간·주말 상시 대기 및 교대근무:** 워라밸 제한
- **하이브리드 환경 복잡성:** 클라우드 전환 가속으로 온프레미스+클라우드 동시 운영 복잡성 증가
- **대규모 고객사 가용성 압박:** ERP 고객사 13만 이상에 대한 인프라 가용성 유지
- **하드웨어 노후화 사이클 대응:** RAID 장애, 스토리지 교체 등 물리적 이슈
- **AI 기반 관제(AIOps) 도입 전환 과도기:** 자동화 도구 학습 부담

# 2. 티맥스소프트

---

## ① B2B IT 솔루션 영업

### 주요 업무 및 책임

- JEUS(WAS), WebtoB, ProObject, AnyLink, OpenFrame 등 미들웨어·플랫폼 솔루션의 기업 고객 영업
- SW 프리세일즈 및 기술 컨설팅: 고객사 아키텍처 분석 → 솔루션 도입 제안
- 금융·제조·공공·통신 등 산업별 고객 맞춤형 제안서 작성 및 발표
- 글로벌 고객 영업 (미국, 일본, 중국, 브라질 등 해외 고객 지원 포함)
- 메인프레임 현대화(OpenFrame) 레퍼런스 확보 및 Winback 영업
- AI 비즈니스 플랫폼(TEM, Continuum AI 제품군) 신규 영업 기회 발굴

### 요구 역량

#### 기술적 역량

- 미들웨어·WAS 기본 개념: WAS, 웹서버, 트랜잭션 처리 구조 이해
- 엔터프라이즈 아키텍처: Java EE, MSA, 클라우드 네이티브 기초
- 공공 및 금융 IT 규격 이해: 전자금융감독규정, 공공 SI 입찰 절차
- 프레젠테이션 도구, 기술 제안서 작성

#### 비기술적 역량

- 고객 신뢰 구축: 레퍼런스(구현 사례) 기반 솔루션 설득 역량
- 장기적 관계 관리: 국내 공공·금융 고객은 수년 단위 계약 구조
- 글로벌 커뮤니케이션 (영어 기반 해외 고객 대응)
- 기술적 깊이: "제품 처음부터 끝까지 경험한 경력직 선호" (업계 평가)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 연·분기별 신규 라이선스 매출 달성률 | % |
| Winback 건수 | 경쟁사 제품 대체 전환 성과 |
| 해외 수주 건수 및 매출 비중 | % |
| 유지보수 갱신율 | 기존 고객 유지율 |
| 프리세일즈 기술 제안 → 계약 전환율 | POC 성공률 |
| 제안서 제출 → 수주 평균 소요 일수 | 일 |

### 직무별 도전과제

- **국내 미들웨어 시장 성숙도:** JEUS 등 국내 점유율 1위지만 시장 자체가 성숙 단계 → 신규 고객 개척 어려움
- **AI 플랫폼 신사업 영업:** TEM, Continuum AI 등 신제품은 레퍼런스 부족, 고객 설득에 시간 소요
- **글로벌 영업 역량 요구:** 해외 시장(일본·브라질 등) 관행·언어 장벽
- **외산 솔루션 대비 포지셔닝:** AWS, Oracle, IBM과의 경쟁에서 기술적 차별성 설명 필요
