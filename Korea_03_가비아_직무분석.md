# 가비아 직무 심층 분석 보고서

**분류:** 국내 | IDC·클라우드
**기준일:** 2026년 6월

> **회사 개요:** IDC·호스팅·클라우드·보안 전문 기업. 핵심 서비스: 가비아 클라우드, 하이웍스(그룹웨어), 코로케이션, 보안관제. 직원 약 700여명. SMB~대기업, 공공기관 대상. CSAP 인증 기반 공공 클라우드 사업 및 AWS 리셀러 사업 병행.

---

## ① B2B IT 솔루션 영업

### 주요 업무 및 책임

- 클라우드(퍼블릭·프라이빗), IDC 코로케이션, 하이웍스(그룹웨어), 보안, 호스팅 서비스의 B2B 영업
- 공공 클라우드 영업: CSAP 인증 환경 기반 공공기관 대상 영업 및 사업 지원
- 신규 파트너(리셀러) 발굴 및 제휴 업무 (AWS 리셀러 사업 포함)
- 고객사 제안서 작성 및 프레젠테이션 지원
- 기존 고객 사업 관리 및 요구사항 대응
- 공공 정보화 사업 입찰 지원 (RFP 분석, 제안 참여)
- 파트너·고객 관계 관리 (Partner Account Manager 역할 포함)

### 요구 역량

#### 기술적 역량

- 클라우드 서비스 지식: CSAP(공공클라우드 보안인증), IaaS/PaaS/SaaS 구조 이해
- 온프레미스 인프라 이해: 서버, 네트워크, 스토리지, 보안 장비 기초 지식
- 공공 정보화 사업 이해: 조달청 나라장터, 공공 SI 프로세스
- AWS, Azure, 가비아 클라우드 서비스 비교 설명 능력
- 보고서 작성 및 프레젠테이션 도구 활용

#### 비기술적 역량

- 다양한 관계자(기술·구매·경영 부서)와의 원활한 커뮤니케이션
- 논리적 사고: 고객 요구사항 분석 → 최적 인프라 솔루션 매핑
- 공공·기업 고객 특성 이해 (긴 의사결정 사이클, 입찰 절차 준수)
- 경력 3년 이상 우대 (공공 클라우드 포지션)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 신규 계약 MRR(월간 반복 매출) 달성률 | % |
| 공공 입찰 참여 건수 및 수주율 | % |
| 신규 파트너 계약 건수 | 건 |
| 고객 유지율 (Churn Rate 역지표) | % |
| 클라우드 전환 고객 수 | 온프레미스→클라우드 마이그레이션 지원 건수 |
| 제안서 제출 후 계약 전환 소요 기간 | 일 |

### 직무별 도전과제

- **가격 경쟁:** 국내외 대형 클라우드(AWS, Azure, KT Cloud 등)와의 가격 및 브랜드 경쟁
- **공공 입찰 복잡성:** CSAP 인증 조건, 조달 절차, 정부 정책 변화에 빠른 대응 필요
- **중소 규모 영업팀으로 다양한 사업군 동시 커버:** 도메인·호스팅·클라우드·보안 병행
- **AWS 리셀러 사업 확대 전략 실행:** 직접 영업과 파트너 채널 영업 병행

## ② EMS/AIOps 소프트웨어 개발자

### 주요 업무 및 책임

- 소프트웨어 기반 가상화 시스템 관리 API 개발 (가비아 클라우드 가상화 레이어)
- 클라우드 콘솔 및 AWS 연동 시스템 개발
- 마이크로서비스 아키텍처(MSA) 기반 솔루션 및 연동 API 개발
- Docker Container 기반 플랫폼 서비스 구현
- 대용량 데이터 처리 및 서비스 API 개발
- 서비스 관리·판매·빌링 시스템 개발
- MLOps 엔지니어링: ML 모델 학습 파이프라인 구성 및 운영 (경력직)
- Research Scientist 지원: 리서치 결과를 서비스화하는 백엔드 개발
- 가상화 원천기술(Hypervisor 등) 개념 검증(PoC) 및 연동 시스템 개발
- 이슈 확인 → 일일 계획 공유 → 코드 리뷰 → 개발 → 현황 업데이트 (일과 사이클)

### 요구 역량

#### 기술적 역량

| 기술/도구 | 수준 |
|---|---|
| Java, Go, Python (서버사이드) | 필수 |
| Docker, Kubernetes, 클라우드 API(AWS SDK) | 필수 |
| 분산 시스템, 대용량 데이터 처리 | 필수 |
| Terraform, vLLM, GPU 클러스터 운영 (MLOps) | 우대 |
| GitLab, Confluence | 필수 (가비아 표준 협업 도구) |
| OpenStack, KVM, Hypervisor 개념 | 우대 |

#### 비기술적 역량

- 자율성: "스스로 일을 찾아서 할 수 있는 사람" (가비아 채용 직무 페이지 명시)
- 협업 능력: 구두·문서·코드 리뷰 등 다양한 방식의 커뮤니케이션
- 판단력: "스스로 판단하고 판단의 근거를 설명할 수 있어야 함" (가비아 채용 직무 페이지 명시)
- 인프라와 개발 양쪽 이해 (풀스택 인프라 개발자로서의 T자형 역량)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 개발 산출물의 완성도와 작업 일정 | **가비아 채용 직무 페이지 공식 명시** |
| 생산성 | 스프린트 내 완료된 이슈/스토리 수 |
| 품질 완성도 | 코드 리뷰 통과율, 결함 발생 건수 |
| 의사소통 기여도 | PR 리뷰 참여, 문서화 기여 |
| 배포 빈도 및 서비스 안정성 | 주기별 지표 |

### 직무별 도전과제

- **가상화/클라우드 기술의 빠른 변화 속도:** OpenStack, Kubernetes 신버전 대응 지속적으로 필요
- **인프라 + 개발 양쪽 전문성 요구:** 순수 SW 개발자보다 넓은 기술 스펙트럼
- **MLOps 신사업 적응:** AI 인프라(GPUaaS, vLLM 서빙) 관련 신기술 빠른 습득 필요
- **소규모 팀 내 광범위한 역할:** "웹 서비스·인프라 시스템·DevOps 등 다양한 요구사항 처리" (가비아 직무 설명 직접 인용)

## ③ IT 인프라 운영 엔지니어

### 주요 업무 및 책임

- 고객사 서버/코로케이션 호스팅 인프라 구성 컨설팅 (비즈니스별 최적 시스템 설계)
- OS 설치 및 애플리케이션 구성 지원 (Apache, PHP, MySQL 등 소스 컴파일 설치)
- 매니지드 서비스 지원: 트러블슈팅, 하드웨어 구성 변경, 기술 지원
- IDC 통합 관제 센터 운영: 가비아 인프라 및 고객 호스팅 서버 모니터링
- 보안관제: 과학기술정보통신부 인정 '보안관제 전문기업' 자격 기반 보안 모니터링
- 호스팅 서비스 장애 분석·개선 (문제 발견 → 부족한 부분 확인 → 개선 방안 도출)
- 시스템 개선을 위한 간단한 스크립트/도구 개발 병행

### 요구 역량

#### 기술적 역량

| 기술/도구 | 수준 |
|---|---|
| Linux 계열 OS (RHEL/CentOS/Ubuntu) | 필수 |
| 하드웨어: RAID 구성, 파티셔닝, 서버 물리 교체 | 필수 |
| Apache, Nginx, PHP, MySQL (소스 설치·관리) | 필수 |
| Bash, Python, PHP, Perl (자동화 스크립팅) | 필수 |
| 고가용성: 이중화 처리 (HA 구성) | 필수 |
| 보안 기초: 방화벽, 권한 관리, DDoS 대응 | 필수 |

#### 비기술적 역량

- 책임감: "IDC에서는 책임감 강한 사람을 찾는다" (가비아 직무 페이지 직접 인용)
- 능동성: "수동적 처리가 아닌 적극적 사고와 판단"으로 업무 처리
- 고객 지향 태도: 신규 고객 유치 및 기존 고객 유지를 위한 최상의 서비스 제공 의식
- 성실성: 야간/주말 근무 가능성 대비 (통합 관제 센터 운영)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 고객 지원 요청의 신속·정확한 처리 여부 | **가비아 직무 페이지 직접 인용** |
| 장애 발생 범위와 정도 | **가비아 직무 페이지 직접 인용** |
| MTTR (장애 평균 복구 시간) | 최소화 목표 |
| 고객 지원 티켓 처리율 및 처리 속도 | SLA 준수율 |
| 시스템 가용성 (서비스 업타임) | % |
| 고객 만족도 (CSAT) | 기술 지원 후 만족도 피드백 |

### 직무별 도전과제

- **사소한 실수로 커다란 문제를 야기할 수 있는 심리적 부담** (가비아 직무 페이지 직접 인용)
- **야간/주말 근무 가능성:** IDC 통합 관제 특성상 24시간 체계
- **고객과의 갈등 발생 가능성:** 장애 상황에서 고객 불만 접수 및 해소
- **기술 다양성 요구:** 수천 개 고객사가 각기 다른 애플리케이션·환경을 운영하므로 폭넓은 기술 지식 필요
