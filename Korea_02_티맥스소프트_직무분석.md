# 티맥스소프트 직무 심층 분석 보고서

**분류:** 국내 | 미들웨어·플랫폼
**기준일:** 2026년 6월

> **회사 개요:** 미들웨어·AI 비즈니스 플랫폼 전문 기업. 핵심 제품: JEUS, WebtoB, ProObject, AnyLink, TEM, Continuum AI. 직원 약 2,000여명. 금융·제조·공공·통신 국내외 고객. 메인프레임 현대화(OpenFrame)와 AI 플랫폼 신사업 동시 추진 중.

## ① B2B IT 솔루션 영업

### 주요 업무 및 책임

- JEUS(WAS), WebtoB, ProObject, AnyLink, OpenFrame 등 미들웨어·플랫폼 솔루션의 기업 고객 영업
- SW 프리세일즈 및 기술 컨설팅: 고객사 아키텍처 분석 → 솔루션 도입 제안
- 금융·제조·공공·통신 등 산업별 고객 맞춤형 제안서 작성 및 발표
- 글로벌 고객 영업 (미국, 일본, 중국, 브라질 등 해외 고객 지원 포함)
- 메인프레임 현대화(OpenFrame) 레퍼런스 확보 및 Winback 영업
- AI 비즈니스 플랫폼(TEM, Continuum AI 제품군) 신규 영업 기회 발굴

### 요구 역량

#### 기술적 역량

- 미들웨어·WAS 기본 개념: WAS, 웹서버, 트랜잭션 처리 구조 이해
- 엔터프라이즈 아키텍처: Java EE, MSA, 클라우드 네이티브 기초
- 공공 및 금융 IT 규격 이해: 전자금융감독규정, 공공 SI 입찰 절차
- 프레젠테이션 도구, 기술 제안서 작성

#### 비기술적 역량

- 고객 신뢰 구축: 레퍼런스(구현 사례) 기반 솔루션 설득 역량
- 장기적 관계 관리: 국내 공공·금융 고객은 수년 단위 계약 구조
- 글로벌 커뮤니케이션 (영어 기반 해외 고객 대응)
- 기술적 깊이: "제품 처음부터 끝까지 경험한 경력직 선호" (업계 평가)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 연·분기별 신규 라이선스 매출 달성률 | % |
| Winback 건수 | 경쟁사 제품 대체 전환 성과 |
| 해외 수주 건수 및 매출 비중 | % |
| 유지보수 갱신율 | 기존 고객 유지율 |
| 프리세일즈 기술 제안 → 계약 전환율 | POC 성공률 |
| 제안서 제출 → 수주 평균 소요 일수 | 일 |

### 직무별 도전과제

- **국내 미들웨어 시장 성숙도:** JEUS 등 국내 점유율 1위지만 시장 자체가 성숙 단계 → 신규 고객 개척 어려움
- **AI 플랫폼 신사업 영업:** TEM, Continuum AI 등 신제품은 레퍼런스 부족, 고객 설득에 시간 소요
- **글로벌 영업 역량 요구:** 해외 시장(일본·브라질 등) 관행·언어 장벽
- **외산 솔루션 대비 포지셔닝:** AWS, Oracle, IBM과의 경쟁에서 기술적 차별성 설명 필요

## ② EMS/AIOps 소프트웨어 개발자

### 주요 업무 및 책임

- TEM(TmaxSoft Enterprise Manager): JEUS, WebtoB, ProObject 등 자사 제품군의 통합 운영 관리 솔루션 개발
- AIOps 기능 개발: 생성형 AI 기반 인시던트 대응, 모니터링 데이터 분석, 이상 감지, 설정 자동화
- Continuum AI 제품군 개발: 비즈니스 프레임워크·엔터프라이즈 매니저·코드 인텔리전스·애플리케이션 트랜스폼
- Java/Go 기반 미들웨어 핵심 엔진 개발 (JEUS 8/9, WebtoB 5/6 등 신버전 개발)
- C/C++/Rust 기반 시스템 소프트웨어 연구 및 구현
- AI/Python 기반 이상탐지·예측 모델 개발 및 서비스화
- 전체 개발 사이클(설계→구현→QA→릴리즈) 참여
- OpenFrame(메인프레임 현대화) 관련 마이그레이션 도구 개발

### 요구 역량

#### 기술적 역량

| 기술/도구 | 수준 |
|---|---|
| Java, Go (백엔드·미들웨어) | 필수 |
| C/C++/Rust (시스템 레벨) | 필수 |
| Python (AI/ML) | 필수 |
| React/Vue (관리 콘솔 UI) | 우대 |
| AI/ML: AIOps 이상탐지 모델, LLM API 연동 | 필수 (신규 역할) |
| Docker, Kubernetes, DevOps 파이프라인 | 우대 |
| Java EE 스펙, JVM 튜닝 | 우대 |

#### 비기술적 역량

- AI 사업 내러티브의 신속한 이해 및 제품화 적용 능력
- 품질·성능·신뢰성 중심 개발 문화 적응
- 글로벌 제품 개발 관점: 다국어·멀티리전 요건 설계
- 빠른 릴리즈 사이클 대응 능력 (신사업 추진 중)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 기능 개발 완료율 및 출시 일정 준수율 | % |
| 제품 버그 발생률 | 출시 후 Critical/Major 결함 건수 |
| AIOps 모델 정확도 | 이상 감지 정밀도(Precision)/재현율(Recall) |
| AI QA 지표 | AI 기능의 오류율·사용자 만족도 |
| 배포 빈도 및 변경 실패율 | DORA 4 지표 |

### 직무별 도전과제

- **레거시 미들웨어와 AI 신기술 통합:** JEUS 등 20년 이상 된 제품에 AIOps 기능 접목의 기술적 복잡성
- **자사 독자 기술 스택:** 오픈소스가 아닌 자사 기술 기반 개발 → 외부 참고 자료 제한
- **빠른 AI 기술 변화 대응:** GenAI·LLM 기술이 빠르게 발전하는 환경에서 제품 경쟁력 유지
- **글로벌 제품 품질 요구:** 해외 금융·공공 고객 대상 제품의 안정성·규정 준수 요건 엄격

## ③ IT 인프라 운영 엔지니어

### 주요 업무 및 책임

- 티맥스소프트 내부 IT 인프라 운영: 서버, 네트워크, 스토리지 관리
- Cloud Architecture 설계 및 운영: 자사 HyperFrame 기반 클라우드 환경 및 퍼블릭 클라우드(AWS/Azure 혼합) 운영
- Network Admin 업무: 사내망·개발환경 네트워크 관리 및 보안 정책 운용
- 개발·QA 환경 인프라 지원: 개발자 빌드/테스트 환경 프로비저닝
- DevOps 인프라 지원: CI/CD 파이프라인, Docker/Kubernetes 클러스터 운영
- 자사 솔루션(JEUS, TEM 등) 배포 및 운영 환경 구성
- 보안 패치, 취약점 관리, 내부 보안 감사 지원

### 요구 역량

#### 기술적 역량

| 기술/도구 | 수준 |
|---|---|
| AWS, Azure, 자체 HyperFrame 클라우드 | 필수 |
| Docker, Kubernetes | 필수 (DevOps 포지션) |
| 라우팅, 스위칭, 방화벽, VPN, 로드밸런서 | 필수 |
| Linux (RHEL/CentOS/Ubuntu), Windows Server | 필수 |
| Ansible, Terraform (IaC) | 우대 |
| 자사 TEM 또는 외부 모니터링 도구 | 우대 |

#### 비기술적 역량

- 자사 개발 환경에 대한 높은 이해도 (JEUS, ProObject 등 솔루션 구동 환경)

### 직무별 도전과제

- **AI 신제품 인프라 요구 급증:** TEM, Continuum AI 개발을 위한 GPU 서버, 대용량 스토리지 수요 급증 대응
- **다중 환경 동시 운영:** 연구(R&D), 개발, QA, 스테이징, 프로덕션 환경 동시 관리
- **자사 솔루션 기반 환경:** 자체 미들웨어(JEUS) 위에서 구동되는 복잡한 의존성 관리
- **보안 규정 준수:** 금융·공공 고객 대상 제품 개발 환경의 보안 인증 유지
