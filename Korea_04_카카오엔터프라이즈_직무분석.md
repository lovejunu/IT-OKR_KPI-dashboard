# 카카오엔터프라이즈 직무 심층 분석 보고서

**분류:** 국내 | AI·클라우드
**기준일:** 2026년 6월

> **회사 개요:** AI·클라우드·B2B SaaS 전문 기업. 핵심 서비스: KakaoCloud(IaaS/PaaS), GPUaaS, KakaoWork. 직원 약 800여명 (2023년 구조조정 후). 대기업·공공기관·금융 대상. 카카오 i AI 기술 기반 엔터프라이즈 클라우드 및 AI 에이전트 사업 집중.

---

## ① B2B IT 솔루션 영업

### 주요 업무 및 책임

- KakaoCloud(IaaS·PaaS·GPU-as-a-Service) 및 B2B SaaS 솔루션 신규 고객 영업
- 카카오클라우드 프리세일즈: 고객사 클라우드 아키텍처 분석 → 최적 서비스 구성 제안
- 기업 대상 AI 솔루션(엔터프라이즈 AI 에이전트, 업무 자동화 솔루션) 영업
- 파트너 채널 관리 및 신규 파트너 발굴
- 국산 클라우드 전환(외산→카카오클라우드) 고객 발굴: 보안성·가격 경쟁력(외산 대비 약 60% 수준) 강조
- GPUaaS 수요 기업(AI 스타트업, 연구기관, 대기업 AI 팀) 대상 영업
- KakaoCloud for Government (공공 클라우드) 공공기관 영업

### 요구 역량

#### 기술적 역량

- 클라우드 서비스 구조: IaaS/PaaS/SaaS, 클라우드 마이그레이션 방법론
- AI 인프라 이해: GPUaaS, 슈퍼컴퓨터, LLM 서빙 인프라 기초
- 기술 제안서 작성: PoC 설계, 아키텍처 다이어그램 작성 능력
- 보안·컴플라이언스: CSAP, 공공 클라우드 보안 인증 이해

#### 비기술적 역량

- 국산 클라우드 차별성 설명 능력: "외산 대비 60% 저렴, 카카오 데이터센터 국내 상주" 등의 가치 제안 숙련
- B2B 엔터프라이즈 영업 경험: 기업·공공기관의 복잡한 의사결정 구조 대응
- 카카오 브랜드 활용 영업: 카카오 생태계(카카오톡·카카오 i) 연계 서비스 제안 역량
- 구조조정 이후 안정화된 클라우드 사업 집중 전략 이해 및 전달

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 신규 클라우드 계약 ARR(연간 반복 매출) 달성률 | % |
| GPUaaS 고객 확보 건수 및 사용량(Usage) 지표 | 건/TB |
| 고객 클라우드 사용량 증가율 (Net Revenue Retention) | % |
| 공공 클라우드 사업 수주 건수 | 건 |
| 프리세일즈 기술 제안 → PoC → 계약 전환율 | % |

### 직무별 도전과제

- **사업 집중도 리스크:** 2023년 대규모 구조조정(인력 30% 감소) 이후 클라우드 사업 집중 → 고객에게 회사 안정성 신뢰 확보가 선결 과제
- **국내 클라우드 시장 경쟁 심화:** AWS, Azure, GCP, NCP(네이버 클라우드), KT Cloud와의 멀티플레이어 경쟁
- **신사업(GPUaaS, AI 에이전트) 영업 레퍼런스 부족**
- **카카오 그룹 이미지:** B2C 이미지 강한 카카오 브랜드를 B2B 엔터프라이즈 신뢰로 전환하는 포지셔닝 과제

## ② EMS/AIOps 소프트웨어 개발자

### 주요 업무 및 책임

- KakaoCloud IaaS 핵심 서비스 개발: 컴퓨팅(인스턴스), 네트워킹, 스토리지, 보안 서비스 설계·구현
- GPU 클라우드 플랫폼 개발: GPUaaS 인프라 제어·스케줄링 시스템 개발
- 컴퓨터 비전·AI 서비스의 백엔드 시스템 개발
- 대용량·실시간 처리를 위한 분산/병렬 처리 시스템 개발
- 대규모 데이터 색인·검색 기술 개발
- ML/딥러닝 학습 데이터 수집·정제·관리 시스템 개발 (MLOps 연계)
- AIOps 관련: KakaoCloud Monitoring 서비스 개발 (멀티 메트릭 수집·시각화·이상탐지·알림)
- MSA 구조 기반 클라우드 서비스 API 설계 및 구현

### 요구 역량

#### 기술적 역량

| 기술/도구 | 수준 |
|---|---|
| C/C++, Python, Go (최소 1개 숙련) | 필수 |
| 자료구조, 알고리즘 (CS 기초) | 필수 |
| 분산 아키텍처, 백엔드 시스템 개발 | 필수 |
| Linux 환경 서비스 설계·개발 | 필수 |
| Kubernetes, Kafka, Hadoop | 우대 |
| GPU 클라우드·ML 백엔드·인덱싱·검색 시스템·MSA 설계 | 우대 |

#### 비기술적 역량

- 논리적·원활한 커뮤니케이션 역량 (채용공고 명시)
- 새로운 기술에 대한 거부감 없는 도전적 자세 (채용공고 명시)
- 하이퍼스케일 환경 (대용량 트래픽·글로벌 분산 아키텍처) 문제 해결 경험
- 카카오 수준의 대규모 서비스 개발 경험 (카카오 i 인프라 내부 활용 및 외부 클라우드 제공 병행)

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 기준 |
|---|---|
| 클라우드 서비스 가용성 (SLA 달성률) | 카카오클라우드 공식 SLA 기준 |
| 기능 출시 속도 (Feature Lead Time) 및 출시 빈도 | 스프린트 단위 |
| 결함 발생률 및 평균 복구 시간 | P0/P1 인시던트 건수·해결 시간 |
| 분산 시스템 성능 지표 | 응답 속도(P99 레이턴시), 처리량(TPS) |
| GPU 클러스터 자원 활용률 | GPUaaS 효율화 |
| AI 기능 정확도 지표 | 모델 성능 지표, 사용자 피드백 |

### 직무별 도전과제

- **하이퍼스케일 클라우드 서비스 수준 요구:** AWS, GCP 수준의 안정성·확장성 달성 필요
- **구조조정 후 소수 팀으로 광범위한 서비스 커버:** 2023년 인력 30% 감소 후 동일한 클라우드 서비스 영역 유지
- **GPU 인프라 신기술 대응:** vLLM 서빙, NVIDIA H100/H200 등 신규 GPU 아키텍처 빠른 대응
- **B2B 고객 요구 다양성:** 금융·공공·제조 등 각 산업별 클라우드 요구사항의 보안·컴플라이언스 대응

## ③ IT 인프라 운영 엔지니어

### 주요 업무 및 책임

- 카카오엔터프라이즈 기업 서비스용 클라우드(Public/Private) 및 On-Premise 인프라 아키텍처 설계·구축·운영
- Site Reliable Engineering(SRE): 서비스 신뢰성·연속성 향상을 위한 지속적 개선 활동
- Linux/Windows OS 기반 시스템 성능 개선 및 지표 관리 시스템 구축·운영
- Infrastructure as Code(IaC): 인프라 및 서비스 환경 자동화 아키텍처 설계·표준화
- 하이브리드·멀티 클라우드 인프라 구축: On-Premise + 퍼블릭 클라우드 연결 환경 운영
- B2B 서비스 엔지니어링: 카카오클라우드 서비스의 고객 기술 문제 해결 (기술지원 엔지니어 역할 포함)
- 서비스 개발 환경 및 업무 생산성 향상을 위한 자동화 도구 개발·운영

### 요구 역량

#### 기술적 역량

| 기술/도구 | 수준 |
|---|---|
| OpenStack, Kubernetes, AWS, GCP, Azure | 필수 (복수 우대) |
| Bash, Python, Golang (스크립팅·개발) | 필수 |
| 분산 아키텍처 서비스 구축·운영 | 필수 |
| Linux/Windows OS 환경 표준화 | 필수 |
| Terraform, Ansible (IaC) | 필수 |
| KakaoCloud Monitoring, Prometheus, Grafana | 우대 |

#### 비기술적 역량

- 논리적이고 원활한 커뮤니케이션 역량 (채용공고 명시)
- 새로운 기술에 대한 거부감 없는 도전적 자세 (채용공고 명시)
- SRE 마인드셋: 운영을 수동적 관리가 아닌 엔지니어링 문제로 접근
- B2B 고객 응대 능력: 기업 고객의 기술 문제를 직접 해결하는 대면·비대면 지원 역량

### 성과 측정 방식 (KPIs / OKRs)

| 지표 | 목표 |
|---|---|
| 서비스 가용성 (SLA 달성률) | 카카오클라우드 공표 SLA 기준 |
| MTTR (인시던트 평균 복구 시간) | 최소화 목표 |
| 인프라 자동화율 (IaC 적용 비율) | "자동화 구현 비율" 추적 |
| 성능 지표 | 시스템 P99 응답 속도, CPU/메모리/GPU 활용률 |
| 인프라 비용 최적화율 | 하이브리드 클라우드 비용 효율 |
| B2B 고객 기술 지원 만족도 (CSAT) | 티켓 해결 시간 |

### 직무별 도전과제

- **대규모 구조조정 이후 소수 정예 팀 운영:** 광범위한 인프라를 적은 인원으로 관리
- **하이브리드·멀티 클라우드 복잡성:** On-Premise, KakaoCloud, AWS/GCP 등 이기종 환경 동시 운영
- **GPU 인프라 급증 대응:** GPUaaS 수요 증가에 따른 GPU 클러스터 확장 및 안정적 운영
- **SRE 문화 내재화:** 전통적 운영 엔지니어 역할에서 SRE·IaC 중심의 엔지니어링 문화로 전환 과도기
